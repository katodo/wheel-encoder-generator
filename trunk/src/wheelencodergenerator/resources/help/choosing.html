Choosing an encoder type

What kind of encoder do you need to use for your project? This quick guide will give you some help deciding.

Start Here

Do you need to measure position?

If so, you need an Absolute encoder. Go to Absolute Encoder below.

Do you need to measure speed?

Then you need an incremental encoder. Go to Incremental Encoder below.

Do your encoder sensors read low...

...when pointed at a white / transparent section of the encoder? If so you might find it easier to invert the encoder.

Absolute

If you need reduced error and single-bit changes between positions (meaning fewer interrupts to deal with, perhaps), you'll want a Gray Code encoder.

If you don't care about those thing so much but you just want to read in your sensor bit values as an integer word to determine position directly, use a Binary Code.

Incremental

Do you need to determine the direction of rotation?

If so, you want a Quadrature track, or you need to place two sensors appropriately for a quadrature signal from a single track. See About rotary encoders: Incremental encoders for more detail.

Do you need to position the wheel to a starting spot?

An index track will mark a single spot on the encoder.
